Item relay

          

            Insert into the shared PostgreSQL items table via another stack service.
            Java forwards X-Request-ID and X-Request-Origin: exercises-java on each relay.
          

          
Rust

          
Calls Rust POST /api/items with JSON {"name": "…"}.

          

            
            
            
          

          

          
Python

          
Calls Python POST /api/items with JSON {"name": "…"}.

          

            
            
            
          

          

          
React Node

          
Calls React Node POST /api/items with JSON {"name": "…"} (direct Postgres).

          

            
            
            
          

          

        


        

          
Stack relays

          

            Multi-hop relays across stack services. Java forwards X-Request-ID and
            X-Request-Origin: exercises-java; each hop logs with the same request id.
          

          
Java → Python → React Node

          

            Creates an item via Python POST /api/relay/react, which relays to React Node
            POST /api/items (direct Postgres).
          

          

            
            
            
          

          

        


        

          
Kafka create user

          

            Publishes a create-user event to Kafka. Java and Rust share consumer group
            exercises-create-user — only one app processes each message and inserts into
            PostgreSQL users. Use List users after a few seconds to confirm.
          

          

            
            
          

          

            
            
            
          

          

        


        

          
Kafka create item (Python)

          

            Publishes a create-item event to Kafka. Only the Python consumer
            (exercises-create-item-python) processes each message and inserts into
            PostgreSQL items. The dashboard request_id is stored in the
            message JSON and X-Request-ID header for log correlation. Use
            List items after a few seconds to confirm.
          

          

            
            
            
          

          

        


        

          
OpenAPI

          

            Interactive REST docs (Swagger UI) for /api/users and /api/items CRUD.
            Dashboard, observability, and Rust routes are excluded. JSON spec: /v3/api-docs.
